## Project Overview

DespensaInteligente is a Flutter mobile app for smart pantry management. It uses Firebase (Auth, Firestore) as backend and is in early MVP stage. The project targets Android, iOS, and Web (macOS/Windows/Linux not configured for Firebase).

**Firebase Project ID:** `despensa-inteligente-c1f9d`

## Common Commands

```bash
# Install dependencies
flutter pub get

# Run the app (requires emulator/simulator or device)
flutter run

# Run static analysis
flutter analyze

# Run tests
flutter test

# Run a single test file
flutter test test/widget_test.dart

# Regenerate Firebase config (if Firebase services change)
flutterfire configure
```

## Architecture

### State Management & DI
Uses **Riverpod** (`flutter_riverpod`) for state management and dependency injection. Providers are defined alongside their services (e.g., `authServiceProvider` in `auth.service.dart`).

### Project Structure
- `lib/main.dart` — App entry point, Firebase init, theme config (dark mode with accent `#cde600`), and `AuthWrapper` that routes based on auth state
- `lib/services/` — Backend service layer (auth, Firestore, recipes). Services are injected via Riverpod providers
- `lib/features/` — Feature-based organization (currently only `auth/presentation/`)
- `lib/firebase_options.dart` — Auto-generated by FlutterFire CLI, do not edit manually

### Auth Flow
`AuthWrapper` (ConsumerWidget in main.dart) listens to `firebaseAuthStateProvider` (a StreamProvider over `authStateChanges()`). Unauthenticated users see `LoginScreen`; authenticated users see `DashboardScreen`.

### Environment Variables
A `.env` file at the project root is required (loaded via `flutter_dotenv`). It holds `OPENAI_API_KEY` and test credentials. **Never commit this file.**

## Git Workflow

Uses simplified GitFlow: `main` is production, `develop` is the integration branch. Feature branches are created from `develop`.

## Key Dependencies

- `firebase_core`, `firebase_auth`, `cloud_firestore` — Firebase stack
- `flutter_riverpod` — State management (note: `provider` is also listed but Riverpod is the primary)
- `go_router` — Declared but not yet wired up; basic navigation used for MVP
- `flutter_dotenv` — Loads `.env` for secrets
